# Galua-Field-library

Библиотека для выполнения арифметических операций в конечном поле Галуа **GF(2⁸)** (256 элементов). Реализованы сложение, вычитание, умножение и деление с использованием таблиц логарифмов и антилогарифмов по модулю неприводимого полинома `0x11B` (x⁸ + x⁴ + x³ + x + 1).

## Особенности

- Чистая реализация на **C99** без внешних зависимостей.
- Поддержка встраивания в ядро Linux (`__KERNEL__`).
- Полный набор полевых операций: `+`, `−`, `×`, `÷`.
- Генерация таблиц логарифмов при первом обращении (ленивая инициализация).
- Модульные тесты, проверяющие аксиомы поля.
- Интеграция с CMake, CTest и генерацией отчётов о покрытии кода (gcovr/lcov).

## Сборка и тестирование

### Быстрый запуск через скрипт

```bash
chmod +x scripts/test.sh
./scripts/test.sh
```

Скрипт создаст каталог `build`, соберёт проект в режиме `Debug`, выполнит тесты и сгенерирует отчёт о покрытии.

### Ручная сборка

```bash
mkdir -p build && cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
make
```

Запуск тестов:

```bash
ctest
```

Генерация отчёта о покрытии (только в Debug):

```bash
make coverage
```

После выполнения в `build` появится `coverage_report.html`.

## Использование

Подключите заголовочный файл и используйте функции:

```c
#include "Galois_Field_256.h"

int main(void) {
    GF256_t a = 0x57;
    GF256_t b = 0x83;

    GF256_t sum = GF256_Add(a, b);    // a + b
    GF256_t diff = GF256_Sub(a, b);   // a - b
    GF256_t prod = GF256_Mul(a, b);   // a × b
    GF256_t quot = GF256_Div(a, b);   // a ÷ b

    return 0;
}
```

### API

| Функция                          | Описание                       |
|----------------------------------|--------------------------------|
| `GF256_t GF256_Add(a, b)`        | Сложение (побитовый XOR)       |
| `GF256_t GF256_Sub(a, b)`        | Вычитание (побитовый XOR)      |
| `GF256_t GF256_Mul(a, b)`        | Умножение в GF(2⁸)             |
| `GF256_t GF256_Div(a, b)`        | Деление a на b (b ≠ 0)         |

Все операции работают с типом `GF256_t`, определённым как `uint8_t` (в пространстве пользователя) или `u8` (в ядре).
